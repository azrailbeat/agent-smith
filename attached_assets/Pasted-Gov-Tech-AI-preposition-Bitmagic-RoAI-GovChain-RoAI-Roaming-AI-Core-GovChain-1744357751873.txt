5. Плюсы такого стека
Резюме: мы строим «шину» ИИ‑навыков (RoAI) поверх Lakehouse и векторного
хранилища, а все юридически значимые артефакты «пришпиливаем» к
GovChain. Стек полностью open‑source + совместим с нац‑инфраструктурой, что даёт
контроль, прозрачность и быстрое тиражирование на все 30+ кейсов.
1. Запись совещания загружается → Kafka topic audio.raw .
2. Temporal запускает transcribe workflow → Whisper GPU‑pod → JSON transcript.
3. Skill meeting_summary (LLM) генерирует протокол, валидирует шаблон.
4. Агент отправляет proposal в Governor‑контракт; голоса фиксируются
(ЭЦП → sig → on‑chain).
5. Hash протокола + CID (IPFS/MinIO) сохраняется в DocumentRegistry smart‑contract.
6. Callback триггерит Tool email_notifier → рассылает утверждённый протокол.
7. Все метрики inference и gas‑cost летят в Grafana FinOps‑борду.
GPU‑pool делится через K8s Device Plugin, LLM‑pods масштабируются HPA‑по QPS.
Besu‑кластер 3 мастера + 2 релеевера; ежедневный anchor‑hash в публичный L2
(Polygon CDK) для внешней проверяемости.
Disaster Recovery: MinIO replication в DR‑регион, periodic snapshot Besu → S3.
Повторное использование: один LLM‑хост — десятки агентов.
Безопасность по умолчанию: все данные внутри кластера, hash → блокчейн.
Юр. сила: PKI‑подпись + immutable ledger.
Гибридный ИИ: секретное — on‑prem KazLM, публичное — GPT‑4 API.
FinOps‑контроль: видно, сколько GPU‑час
